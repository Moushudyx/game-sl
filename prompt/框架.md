这是软件的最初设计

使用 tauri + react + ant-design 的技术制作一个简单的游戏存档备份/复原的软件

目前的设计是：仅支持 Windows 平台的存档备份，不考虑其他平台；复原功能先不做，但是代码里需要留下改造的余地

1、获取用户存档路径模块：

路径模块应该做成可配置化的

1-1、游戏一般有固定的存档路径，分为两种，一种写在用户文件夹里，一种是 Steam 存档

1-1-1、用户文件夹：比如说“黑夜君临”的存档位置是“{用户文件夹}\AppData\Roaming\Nightreign”，程序运行时需要拼接用户文件夹的位置得到完整路径
1-1-2、Steam 存档：比如说“怪物猎人荒野”的存档位置是“{Steam安装目录}\userdata\{用户的steamUID}\2246340\remote”其中“2246340”就是游戏ID；
这里还会需要一个获取用户 SteamID 的功能，我会在后面说明

1-2、打开软件时会自动扫描以获取 用户文件夹、Steam安装目录 等信息；进一步通过“{Steam安装目录}\userdata”下的文件夹名称获取所有可用的 用户steamUID（页面上应当有一个下拉框，用户可以自行选择使用哪个 steamUID，默认选中第一个）

1-3、我的设想是最终会使用一个字符串，形如“{AppData}\Roaming\Nightreign”或者“{Steam}\userdata\{SteamUID}\{SteamGID}\remote”，在 tauri 的后端拼接为完整路径；
这样的话页面上可以添加一个编辑功能，用户可以自行编辑存档位置；
软件在初始化运行时会在软件目录中创建一个 game-sl 文件夹，然后将内置的游戏存档路径配置写进去（config.json）；
游戏存档路径配置需要包含“游戏名称”“游戏存档路径”“游戏图片URL”“上次备份时间”，格式为 `{ name: string, path: string, icon: string, lastSave: number }[]`
整个 config.json 的格式是 `settings: { /* 配置项 */ }, games: { name: string, path: string, icon: string, lastSave: number }[], version: number` 其中 version 为 1，因为配置文件格式可能在后续迭代中变更

2、备份存档模块：

2-1、使用 7Z 算法将文件夹打包成 .7z 文件

2-2、文件名固定使用“{游戏名称}-Backup-{年月日}-{时分秒}.7z”的格式，其中 {游戏名称} 可以用户自行配置，时间相关内容取开始备份的时间戳；
示例 “怪物猎人荒野-Backup-20251219-224805.7z” “黑夜君临-Backup-20251218-212952.7z”；

2-2-1、备份存档时用 ant-design 的弹框组件让用户输入存档备注，可以不输入任何备注信息；
2-2-2、如果有备注信息，备份存档时会在 软件目录\game-sl\backup 文件夹中额外添加一个 “{游戏名称}-Backup-{年月日}-{时分秒}.txt” 文件，用于记录存档的备注信息；
2-2-3、这些文件存在 软件目录\game-sl\backup 文件夹中

2-3、用户可以在界面上看到某款游戏的所有备份

2-3-1、软件在用户操作时读取 软件目录\game-sl\backup 文件夹所有以 “{游戏名称}-Backup” 开头的 .7z/.zip 文件（需要兼容其他地方下载到的存档格式）
2-3-2、读取到后继续读取文件名上的“{年月日}-{时分秒}”如果读取不成功则展示未知日期，备份列表按时间排序（没有读取到时间的备份文件按文件的最后更新日期参与排序，但仍展示为未知日期）
2-3-3、用户可以看到与存档文件同名的 .txt 内的文本作为备注信息（没有此文件视为没有备注信息），有编辑按钮，备注信息可以修改
2-3-4、除了编辑按钮外还有删除、复原按钮

2-4、删除备份时需要用 ant-design 的弹框组件二次确认；删除存档时需要一并删除同名的 .txt 文件

3、复原存档模块（现阶段先不做）：

3-1、用户在备份文件列表中点击复原按钮可以实现存档复原，复原时需要用 ant-design 的弹框组件二次确认

3-2、此功能需要做成可配置：复原存档时在 软件目录\game-sl\extra-backup 下额外生成备份，因为复原存档属于危险操作，会覆盖现有存档，额外备份不展示在备份列表中

3-3、额外备份生成完成后，删除存档位置的文件/文件夹（丢进回收站），然后解压备份，写入文件

4、界面设计

4-1、界面分为主界面、配置界面、软件信息界面

4-2、主界面以卡片形式展示，一个游戏对应一个卡片，展示游戏名、游戏图片、上次备份时间；卡片上有备份、查看备份、删除游戏、前移、后移按钮

4-2-1、查看备份按钮点击后弹框展示所有备份（逻辑同 2-3）
4-2-2、删除游戏则是删除此游戏的配置信息，同时删除所有备份文件；此按钮双击触发，且需要用 ant-design 的弹框组件二次确认
4-2-3、前移、后移则是控制卡片展示顺序

4-3、配置界面，展示用户配置

复原存档时额外生成备份：默认为是

4-4、软件信息界面，纯信息展示，其中需要有一个跳转 GitHub 的超链接
